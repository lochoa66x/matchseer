# Phase 0 Experience Direction

## Goal

MatchSeer should feel like a modern matchday companion: quick, playful, visual, and easy to use without making people fill out a bunch of inputs.

The experience should be fun first, analytical second, and complicated never.

## First Impression

The first screen should immediately show useful match content:
- Today’s matches
- Upcoming matches
- Live or final scores
- Forecast highlights
- A quick language switcher

Do not start with:
- A marketing landing page
- A giant form
- A dashboard full of controls
- A wall of statistics

## Interaction Principles

Primary interactions:
- Tap a match
- Tap a team
- Tap a player
- Switch tabs
- Select from chips
- Search only when needed
- Share a card

Avoid:
- Long forms
- Too many filters
- Dense tables on mobile
- Settings-heavy flows
- Requiring user accounts for basic use

## Visual Feel

The visual system should be:
- Modern
- Bright but not childish
- Sporty without looking like a sportsbook
- Playful without losing trust
- Mobile-first
- Fast to scan

Suggested UI elements:
- Match cards
- Forecast cards
- Team edge meters
- Chaos meter
- Weather chips
- Player comparison chips
- Small trend charts
- Simple segmented controls
- Clear language toggle

## Main Screens

### Home

Purpose:
- Show what matters today.

Modules:
- Matchday header
- Live/upcoming match list
- Featured forecast
- Chaos match of the day
- Top player spotlight

### Match Detail

Purpose:
- Make one match feel exciting and understandable.

Tabs:
- Forecast
- Teams
- Players
- Weather
- Review after match

### Compare

Purpose:
- Let users compare teams or players without feeling like they are filling out a form.

Interaction:
- Pick from suggested teams or players first.
- Search is available, but not required.
- Use chips, cards, and quick swaps.

## Forecast Card Pattern

A forecast card should include:
- Teams
- Projected score range
- Win/draw/win projection
- Confidence
- Chaos meter
- Three key reasons
- One playful tone line
- Required disclaimer

The forecast should be scannable in under ten seconds.

## Fun Without Clutter

Fun features that fit the vibe:
- Chaos meter
- Weather mood
- Tactical edge badge
- Upset cloud
- Player spark badge
- Post-match “forecast receipt”

Keep advanced stats available, but tucked behind details.

## Mobile Rules

Mobile is the main product.

Rules:
- No tiny tables.
- No overloaded navigation.
- Buttons should be easy to tap.
- Cards should stack cleanly.
- Text should be short and readable.
- Key actions should be reachable with one thumb.

## Design Direction For Sprint 1

Sprint 1 should create:
- A polished home screen
- A match detail screen
- A forecast card
- A team comparison module
- A player comparison module
- Language switching

All of it should work with sample data before we connect real data.
